Contributing to Opticals Shop

Thanks for wanting to contribute! A few simple guidelines to make collaboration smoother.

Getting started
- Fork the repo and create feature branches from `main`.
- Branch naming: `feature/<short-description>`, `fix/<short-description>`, `chore/<short-description>`.

Local development
- Backend:

```powershell
cd "C:\Users\admin\Desktop\Opticals solution\starter-backend"
npm install
npm start
```

- Frontend:

```powershell
cd "C:\Users\admin\Desktop\Opticals solution\starter-frontend"
npm install
npm run dev
```

Commits & PRs
- Use clear, imperative commit messages, e.g. `feat(customers): add DOB field`.
- Open a pull request against `main` describing the change, motivation, and testing steps.

Code style
- Keep code readable and small PRs when possible. Add tests for new functionality.

Security
- Do not commit secrets or tokens. Use environment variables for credentials.

Issues
- Use the provided issue templates (bug report, feature request) when filing issues.

Thank you — we appreciate your contributions!
